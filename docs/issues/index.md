# 问题记录

这里收录了仓库实现过程中的问题记录、设计笔记与排查过程。

- 建议从侧边栏选择具体条目阅读
- 如果只是快速定位主题，可以在文件名中搜索关键词

## 状态枚举

为避免「已修复/已优化/待修复/待优化」等口径不一致，`docs/issues/**` 统一使用以下状态枚举：

- `已解决`：问题已处理完成（包含修复/优化/补测/文档同步等），并已有回归验证或明确结论。
- `待解决`：问题已确认存在，待排期处理。
- `待设计`：问题已确认，但方案/边界未定，需要先完成设计/决策。
- `按设计`：行为符合当前设计，不计划处理；如需变更请另开条目说明目标行为与兼容策略。

### 书写约定

- 标题统一使用：`## N. 标题（状态：<枚举>）`
- 可选补充：`（级别：Major；状态：已解决）`、`（状态：已解决；备注：仍可细化）`
